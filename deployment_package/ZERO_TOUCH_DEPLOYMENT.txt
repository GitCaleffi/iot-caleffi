STEP 1: CREATE MASTER DEVICE (ONE TIME SETUP)
==============================================

1.1 Setup ONE Raspberry Pi completely:
    - Flash fresh Raspberry Pi OS
    - Install barcode scanner system
    - Configure all settings
    - Test everything works perfectly

1.2 Commands for master setup:
    cd deployment_package
    ./install.sh
    # Test barcode scanning works
    # Verify IoT Hub connection
    # Confirm LED feedback works

1.3 Shutdown master Pi:
    sudo shutdown -h now


STEP 2: CREATE MASTER SD CARD IMAGE (ONE TIME)
===============================================

2.1 Remove SD card from master Pi
2.2 Insert into computer with SD card reader
2.3 Create image file:
    
    # Linux/Mac:
    sudo dd if=/dev/sdX of=barcode-scanner-master.img bs=4M status=progress
    
    # Windows (use Win32DiskImager):
    Select SD card → Read → Save as barcode-scanner-master.img

2.4 Compress image for distribution:
    gzip barcode-scanner-master.img
    # Creates: barcode-scanner-master.img.gz


STEP 3: MASS PRODUCTION (2000 DEVICES)
=======================================

3.1 AUTOMATED SD CARD FLASHING:

    Option A - Industrial SD Card Duplicator:
    - Load master image
    - Insert 10-50 SD cards simultaneously  
    - Flash all cards in parallel
    - Repeat until 2000 cards done

    Option B - Script-based flashing:
    #!/bin/bash
    for i in {1..2000}; do
        echo "Flashing SD card $i of 2000..."
        dd if=barcode-scanner-master.img of=/dev/sdX bs=4M
        echo "Card $i complete - insert next card"
        read -p "Press Enter when next card inserted..."
    done

3.2 WHAT HAPPENS ON FIRST BOOT (AUTOMATIC):
    - Pi boots from cloned SD card
    - System detects it's a new device (different MAC address)
    - Generates UNIQUE device ID automatically
    - Registers with IoT Hub using new device ID
    - Starts barcode scanning service
    - GREEN LED = Ready to scan

                                DEVICE UNIQUENESS
================================================================================

HOW EACH DEVICE GETS UNIQUE ID (AUTOMATIC):
===========================================

✅ Hardware-based ID generation:
   - Uses CPU serial number: /proc/cpuinfo
   - Falls back to MAC address: /sys/class/net/eth0/address
   - Each Pi has different hardware = different device ID

✅ Example device IDs generated:
   - scanner-a1b2c3d4 (Pi #1)
   - scanner-e5f6g7h8 (Pi #2) 
   - scanner-i9j0k1l2 (Pi #3)
   - ... (2000 unique IDs)

✅ Automatic registration:
   - Each device registers itself with unique ID
   - No conflicts or duplicates possible
   - Your backend receives 2000 different device registrations
